# Story Definition of Done Checklist

Developer self-check. Enforced by enforce-dod.py hook.

## 1. Requirements Met
- [ ] All functional requirements implemented
- [ ] All acceptance criteria addressed
- [ ] Edge cases handled

## 2. Coding Standards
- [ ] Follows coding-standards.md
- [ ] Files per source-tree.md
- [ ] Tech stack per tech-stack.md
- [ ] No linting errors

## 3. Testing
- [ ] Unit tests written
- [ ] Integration tests where applicable
- [ ] All tests pass
- [ ] Coverage meets threshold

## 4. Functionality
- [ ] Manually verified
- [ ] Edge cases tested
- [ ] Error handling user-friendly

## 5. Story Administration
- [ ] Task checkboxes marked
- [ ] File List updated
- [ ] Change Log updated
- [ ] Dev Agent Record filled

## 6. Build & Dependencies
- [ ] Build succeeds
- [ ] Linting passes
- [ ] No unapproved dependencies
- [ ] No hardcoded secrets

## 7. Documentation
- [ ] Non-obvious code commented
- [ ] New env vars in .env.example
