# Story-Ready Checklist

Engineering Lead validates before spawning any developer.

## Required Fields (ALL must be present)
- [ ] Story title
- [ ] User story (As a / I want / So that)
- [ ] Acceptance criteria (Given-When-Then)
- [ ] Implementation tasks (numbered with S/M/L)
- [ ] API contracts referenced (section citations)
- [ ] DB schema referenced (table names)
- [ ] Edge cases (minimum 3)
- [ ] Security notes
- [ ] Testing notes
- [ ] Epic link
- [ ] Priority

## Brownfield Additional (if applicable)
- [ ] Existing code affected
- [ ] Regression risk (H/M/L)
- [ ] Rollback procedure
- [ ] Feature flag recommendation
